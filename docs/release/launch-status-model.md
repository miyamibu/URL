# Launch Status Model

## Goal
Separate repo-local readiness from external staging, internal testing, public launch, and stop conditions.

## Status Definitions

| Status | Definition | Required evidence | Who performs it | What Codex may do | What Codex must not do | Exit criteria | Stop conditions |
|---|---|---|---|---|---|---|---|
| `REPO_GO` | Repo implementation, docs, scripts, tests, builds, and release hygiene are internally ready. | Core checks pass; MCP contract pass; release hygiene pass; secret scan has no real secrets. | Codex or maintainer. | Edit repo files, run local checks, create docs/scripts. | Commit, push, deploy, submit, enter production secrets. | Manual-only external work remains. | Any internal test/docs/script/config blocker. |
| `STAGING_GO` | Staging MCP/web environment is deployed and safe to exercise with non-production or staged credentials. | HTTPS endpoint, auth, disabled-by-default behavior, smoke tests, logs without raw prompt/body/token. | Maintainer with hosting access. | Provide commands, smoke scripts, expected results. | Deploy or enter secrets unless explicitly instructed. | Staging smoke tests pass and rollback is known. | Noauth data, wrong environment, real secrets in logs, raw body output. |
| `INTERNAL_TEST_GO` | Android internal testing and iOS TestFlight builds are available to internal testers and pass manual QA matrix. | Play internal track/TestFlight build, version evidence, device QA artifacts, privacy/store form review. | Maintainer or release owner. | Provide build commands, checklists, QA matrix. | Upload to stores or manage testers without explicit instruction. | Pixel 9a and iPhone QA pass or issues are triaged. | Crash, data loss, wrong app ID, AI UI visible with flag off, privacy mismatch. |
| `LAUNCH_READY_REPO` | Repo contains all launch-operation artifacts and executable readiness checks; remaining work is external/manual. | `check_launch_readiness.sh` pass; docs/checklists exist; core REPO_GO checks still pass. | Codex or maintainer. | Add/update launch docs and scripts, verify local checks. | External deploy/submission/secret entry. | Manual operators can perform staging/internal/launch without missing repo instructions. | Missing checklist, invalid reference, generated root archive, actual secret committed. |
| `LAUNCH_GO` | Public release can be started from external consoles. | `STAGING_GO` + `INTERNAL_TEST_GO` + privacy/store/OpenAI approvals + final owner approval. | Release owner. | Prepare final report only. | Click final publish/submit without explicit current approval. | Owner approves launch and external consoles are ready. | Reviewer blocker, privacy mismatch, failed live smoke, unresolved incident. |
| `NO_GO_INTERNAL` | Repo-local issue remains. | Failing test, broken script, missing doc, unsafe default, or contradiction. | Codex or maintainer. | Fix repo issue and re-run validation. | Reclassify external without evidence. | Internal blocker fixed. | Same blocker persists. |
| `BLOCKED_EXTERNAL` | Repo is ready but external account/service/permission state blocks progress. | Error from Apple/Google/OpenAI/Supabase/Vercel or missing owner permission. | Release owner. | Document blocker and exact next manual action. | Pretend external work is complete. | External access/action completed. | External account remains inaccessible. |
| `NOT_VERIFIED` | Environment prevents determining whether the issue is internal or external. | Command logs show environment failure or missing tooling. | Codex or maintainer. | Provide bounded retry/diagnosis plan. | Claim readiness without evidence. | Verification rerun passes or is classified. | Ambiguous failure remains. |
